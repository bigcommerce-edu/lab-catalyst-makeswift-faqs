# Changelogs

Each progressive history release of this lab has a changelog entry here, named `<version>.md` (the project version, plain semver — separate from the base-framework version).

The project version is recorded in `package.json`'s `version` field and tagged on the tip of the corresponding progressive history.

## Entry format

```markdown
# <version>

_Released <date> · based on <base-framework> <framework-version>_

## Summary
One or two sentences describing the release.

## Changes
- Bullet points of notable changes (features, refactors, framework/dependency upgrades).
```
